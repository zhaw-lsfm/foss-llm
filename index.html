OpenSource & LLM

“Openness” is a spectrum

More open ↑

  • Fully Open Source (fully reproducible)
  • Open Weights + Code (auditable methodology, but not data)
  • Open Weights (runnable, but not verifiable)
  • Proprietary (black box)

More closed ↓

Proprietary (ChatGPT, Claude, Gemini): - Complete black box - you only get API access - No insight into how it works, what data it was trained on, or how decisions are made - You’re entirely dependent on the vendor

Open Weights (Llama 3.3, DeepSeek, Mistral, Qwen): - The trained model weights are released - you can download and run them locally - BUT: Training code and training data typically NOT included - You can USE it, but can’t fully verify or reproduce how it was created - This is where MOST “open” models sit today - For practical teaching purposes: This is sufficient! You get local control.

Open Weights + Code (Some research releases): - Weights + the methodology/code used for training - You can understand and audit HOW it was trained - BUT: Training data usually not included (licensing issues, competitive advantage) - Example: Some academic lab releases - Closer to true “open source” philosophy - transparent and auditable

Fully Open Source (OLMo, BLOOM - rare!): - Weights + Training Code + Training Data - everything - Fully reproducible and auditable - This is the gold standard but extremely rare (training data is expensive, complex licensing) - Most similar to traditional open source software

You don’t need the absolute best model

Practical Demo

Step 1: Get a Runtime Environment

You need software that can run the model on your machine

  • Downloads the model files
  • Manages memory and processing
  • Provides an interface (API, chat, etc.)

Popular options:

Ollama (ollama.ai) - Recommended for beginners

  • Command-line tool, very simple
  • Works on Mac, Linux, Windows
  • Easy model management: ollama pull deepseek-r1
  • Built-in API server
  • Free and open source

LM Studio (lmstudio.ai)

  • Graphical interface, very user-friendly
  • Great for non-technical users
  • Model browser built-in
  • Works on Mac, Windows, Linux
  • Free

GPT4All

  • Desktop app with GUI
  • Good for absolute beginners
  • Limited model selection but curated
  • Free and open source

Jan.ai

  • Desktop app, modern interface
  • Local-first philosophy
  • Good privacy focus
  • Free and open source

For this workshop, we’ll focus on Ollama - it’s the most flexible and widely used, but any of these will work for getting started.

Step 2: Choose and Download a Model

Models vary by:

  • Size (1GB to 100GB+)
  • Capability (general chat, coding, reasoning)
  • Language (multilingual vs English-focused)

Start with a small, general-purpose model

How to choose:

For beginners, recommend: - DeepSeek-R1:1.5B or 7B - Excellent reasoning, good for teaching tasks, reasonable size - Llama 3.2 3B - Fast, capable, widely supported - Qwen 2.5 7B - Strong multilingual, good for international contexts - Phi-4 - Microsoft’s efficient small model, surprisingly capable

Size considerations: - 1-3B parameters: Fast, runs on most laptops, good for basic tasks (2-6GB RAM needed) - 7-8B parameters: Sweet spot for teaching - good quality, reasonable speed (8-16GB RAM) - 13B+ parameters: Better quality, but slower and needs more RAM (16GB+ minimum)

With Ollama:

ollama pull deepseek-r1:1.5b    # Small, fast
ollama pull llama3.2:3b          # General purpose
ollama pull qwen2.5:7b           # Multilingual

With LM Studio: - Open the model browser - Search for “deepseek” or “llama” - Look for models marked with your RAM capacity - Click download

Key point: You can always download more models later. Start small, test it out, then try larger models if needed. Models are just files - you can delete them if you don’t like them.

Step 3: Run Your First Query

Now you can interact with the model

  • Start the model/server
  • Send it a prompt
  • Get a response

That’s it! You’re running AI locally.

With Ollama (command-line):

Start a chat session:

ollama run deepseek-r1:1.5b

Then just type your question:

>>> Generate 3 multiple choice questions about photosynthesis

Or use it as an API:

curl http://localhost:11434/api/generate -d '{
  "model": "deepseek-r1:1.5b",
  "prompt": "Summarize this: [paste text]"
}'

With LM Studio (GUI): 1. Select your downloaded model from the left sidebar 2. Click “Load Model” 3. Type your prompt in the chat interface 4. Hit enter

With GPT4All or Jan.ai: - Similar - select model, type in chat box, get response

First prompts to try: - “Generate 5 practice questions about [your subject]” - “Summarize this text in simple language: [paste text]” - “Give feedback on this student paragraph: [paste text]” - “Explain [concept] at a high school level”

What’s happening behind the scenes: - The model file is loaded into RAM - Your text is converted to numbers (tokens) - The model predicts the next token, over and over - Results are converted back to text - All happening on your machine, no internet needed (after download)

Key insight: Once you get to this point, you realize it’s not magic - it’s just software running locally. You have full control.

The Three Enhancements

RAG

Retrieval Augmented Generation

Give LLM access to YOUR documents

Example: IUNR course materials, research papers

MCPs

Model Context Protocol

Connect LLM to your tools

Example: Field data, GIS, email, calendar

Personas

System Prompts

Shape how LLM behaves

Example: “Teaching assistant for environmental science”

These three enhancements make local LLMs powerful. Don’t go too deep technically - just explain what each does and give concrete examples relevant to teaching. RAG is especially important for their use case.

Why Local Makes Sense for Teaching

  • Student data is sensitive (GDPR, privacy)
  • Repetitive tasks (grading, feedback)
  • Custom course materials (RAG)
  • One-time setup, ongoing value
  • No per-query costs

Connect the dots: student data + GDPR = must be local. Repetitive tasks + no per-query cost = economically viable. This is why local makes sense specifically for teaching.
